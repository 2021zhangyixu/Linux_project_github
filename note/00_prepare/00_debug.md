# 应用程序出现故障

1. 在开发板上，启用核心转储（core dump） 

   ```shell
   ulimit -c unlimited
   ```

    

2. 在 PC 端编译程序时，交叉编译工具链加上 -g 选项。

  ```shell
  $(CROSS_COMPILE)gcc -g -O0 -o $(APP_TEST) $(APP_TEST).c
  ```

  

3.  在开发板中调用 gdb 工具，后续调用 bt 命令即可查看崩溃点的函数调用关系。从下图可知，我们是在 `strtol ()` 函数崩溃的，该处发生了 `Segmentation fault`（段错误）
